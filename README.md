## 概要

BotW.py(ブレスオブザワイルドのミニゲームの再現)とrandommaze.py(迷路がランダム生成されるバージョン)の説明。

## 開発環境・ツール

- 言語: Python 3
- 物理エンジン: PyBullet
- 開発補助: GitHub Copilot（デバッグ、コマンド検索に使用）

## 遊び方

盤面をキー操作で傾けることによってボールを転がしてゴールへと運ぶゲームになっている。

BotWは最後にボールを跳ね上げてゴールさせる必要があり、迷路の道中の赤いボックスを集めたりより早くゴールすることでスコアを上げることができる。

randommazeではゴールの場所が赤く示されているのでそこに触れるだけでゴールでき、一つの盤面をクリアした後さらに大きなサイズの盤面をランダム生成できるので、無限に暇つぶしができる。

1. 共通操作

    ・"h" -> 盤面の左側が下がる

    ・"k" -> 盤面の右側が下がる

    ・"j" -> 盤面の手前側が下がる

    ・"u" -> 盤面の奥側が下がる

    ・"r" -> 盤面の傾きやボールの位置、時間のカウントをリセットする

    ・"スペース" -> 盤面が傾く速度を上げる

2. BotWについて

    ・"c" -> カメラを切り替える

3. randommazeについて

    ・"n" -> ゴール後に押すことで、よりサイズの大きな盤面を作成する

## プログラム作成時に工夫した点

1. 共通点

    ・盤面は空中に浮かせたかったので質量を0にして力の影響を受けなくなるようにした。運動方程式を考えると加速度は定まらないような気もするが、pybulletでは加速度0になるようだった。

    ・盤面は力の影響を受けないので、盤面を動かすためにp.resetBasePositionAndOrientationを使用した。この際1stepあたりの盤面の移動量が大きくなるとボールがすり抜けてしまうため1stepの時間幅を小さくするなどして対応した。

    ・p.configureDebugVisualizer(p.COV_ENABLE_RENDERING, 0/1)を活用して盤面の生成の時間を短縮した。

2. result_BotWについて

    ・盤面のブロックの座標は原作を参照しながら手打ちした。

    ・原作と違う点として、道中に赤いスコアブロックを配置した。これらのブロックは盤面と一緒に動き、ボールが触れると消えるようにした。

3. result_randommazeについて

    ・参考欄のサイトを参考にして迷路の盤面をランダム生成するようにした。その際、カメラの位置や1stepあたりの回転角度などがサイズに応じて適切に設定されるようにした。

    ・スタート位置は左下に固定したが、できるだけゴールまでの道のりは長い方がいいと考えたため、BFSによりスタートから最も遠い点を求めてゴール地点とした。

    ・サイズが大きくなるとp.createMultiBodyの実行時に物体が正しく表示されないことが分かったので、迷路の情報を持つ二次元配列の行ごとにp.createMultiBodyを適用して、よりサイズの大きい盤面でも正しく表示されるようにした。それぞれの物体はキーボード入力によって一緒に動くようにしている。

    ・次の盤面に移行する際にボールだけは新たに作り直す必要がないため、ボールにはp.resetBasePositionAndOrientation、それ以外にはp.removeBodyを適用（した上で新たなものを定義）するようにした。

    ・make_maze関数だけを先に何回か実行して良さそうな迷路を保存しておく（同じ迷路を別の人が解けるようになる）こともできる。
